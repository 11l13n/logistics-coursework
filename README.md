# Логистическая информационная система

Веб-приложение для курсовой работы по теме планирования перевозок и учета путевых листов. Проект сделан для небольшой логистической компании: диспетчер принимает заявки от клиентов, подбирает машину и водителя, создает маршрут, а водитель потом видит свой рейс и заполняет фактические данные по поездке.

Приложение состоит из frontend на React и backend на Express. Данные хранятся в PostgreSQL, работа с базой сделана через Prisma.

## Основная идея

Основная задача системы - собрать в одном месте заявки, водителей, автомобили, маршруты и путевые листы. В обычной работе диспетчеру приходится отдельно смотреть свободные машины, грузоподъемность, водителей и даты рейсов. В этом проекте часть этих действий вынесена в приложение.

Диспетчер создает заявку на перевозку, указывает клиента, груз, адреса и дату доставки. После этого можно перейти к планированию маршрута. На этом этапе система предлагает подходящую машину и водителя, считает примерную дистанцию и плановый расход топлива.

Водитель работает через отдельный кабинет. Он видит назначенные маршруты, может начать рейс, завершить его и внести фактический пробег и расход топлива. Эти данные потом используются в отчетах.

## Установка

Сначала нужно установить зависимости отдельно для backend и frontend:

```bash
cd logistics-coursework
npm install --prefix server
npm install --prefix client
```

## Создание базы данных PostgreSQL

Пример создания базы через `psql`:

```sql
CREATE DATABASE logistics_coursework;
```

После этого надо создать файл `.env` в папке `server`. Можно скопировать пример:

```bash
cd server
cp .env.example .env
```

Пример содержимого `server/.env`:

```env
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/logistics_coursework?schema=public"
JWT_SECRET="change-this-secret"
PORT=4000
HOST=127.0.0.1
CLIENT_URL="http://localhost:5173"
```

Для frontend тоже есть пример переменных:

```bash
cd ../client
cp .env.example .env
```

Пример `client/.env`:

```env
VITE_API_URL="http://127.0.0.1:4000/api"
```

## Подготовка Prisma

Из корня проекта:

```bash
npm run prisma:generate
npm run prisma:migrate
npm run prisma:seed
```

`prisma:seed` добавляет тестовых пользователей, водителей, автомобили, клиентов, заявки и несколько маршрутов. После этого приложение уже можно открывать и смотреть основные сценарии.

## Тестирование

Unit-тесты находятся в папке `server/tests`. Они проверяют серверную логику без подключения к настоящей базе данных, поэтому PostgreSQL для их запуска не нужен.

Запуск из корня проекта:

```bash
npm test
```

В тестах проверяются авторизация, роли пользователей, создание и редактирование заявок, подбор водителя и автомобиля, создание маршрута, путевые листы, изменение статусов рейса, отчеты и отдельные функции для работы с датами, адресами и маршрутом.

## Роли пользователей

### Администратор

Администратор может управлять пользователями, водителями, автомобилями, клиентами, заявками, маршрутами и путевыми листами. Эта роль нужна в основном для настройки системы и контроля данных.

### Диспетчер

Диспетчер работает с заявками и планированием перевозок. Он добавляет клиентов, создает заявки, выбирает маршрут, назначает водителя и автомобиль, а также смотрит отчеты.

### Водитель

Водитель видит только свои рейсы и путевые листы. В личном кабинете он отмечает начало и завершение рейса, после поездки вносит фактический пробег и расход топлива.

## Подбор водителя и автомобиля

В проекте есть простой модуль рекомендации. Он не использует внешние сервисы, а работает по правилам внутри backend.

При подборе учитывается:

- вес груза;
- грузоподъемность автомобиля;
- статус автомобиля;
- статус водителя;
- занятость водителя и машины на выбранную дату;
- примерный расход топлива.

Например, если груз весит 1200 кг, система не будет предлагать автомобиль с меньшей грузоподъемностью. Также не выбираются машины со статусом `BUSY`, `REPAIR` или `INACTIVE`.

После проверки система возвращает подходящий вариант и короткое объяснение, почему выбран именно этот водитель и автомобиль.

## Путевые листы

Путевой лист создается для маршрута и хранит плановые и фактические показатели. В нем можно посмотреть:

- номер путевого листа;
- маршрут;
- водителя;
- автомобиль;
- плановую дистанцию;
- плановый расход топлива;
- фактический пробег;
- фактический расход топлива;
- статус рейса.

Это нужно, чтобы после выполнения перевозки можно было сравнить план и факт.
